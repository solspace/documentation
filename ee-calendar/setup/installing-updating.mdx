# Installing & Updating

## Requirements

Solspace Calendar has mostly the same minimum requirements as listed on the [ExpressionEngine Requirements](https://docs.expressionengine.com/v7/installation/requirements.html) page:

- ExpressionEngine 7.x
- PHP 8.0-8.3
- [PHP iconv](http://php.net/manual/en/book.iconv.php) enabled
- MySQL 5.6 or newer

:::danger
#### Calendar currently only supports PHP 8.0 through 8.3.
- PHP 8.4+ is not fully supported at this time because adding full compatibility would require dropping support for PHP 8.0 and 8.1.
- Calendar 5.x no longer supports PHP 7.4. If your site requires PHP 7.4, please use Calendar 4.x instead.
:::

## Installing Instructions

<VerticalStepWrapper>
  <StepMarkdown stepTitle="Unzip the Download Package">
    - Open the **EE7** folder.
    - Copy the `calendar` folder to your EE site's `./system/user/addons/` directory.
    - Within the `themes` folder, copy the `calendar` folder to your EE site's `./themes/user/` directory.
  </StepMarkdown>
  <StepMarkdown stepTitle="Install the Add-on">
    - Go to the _Add-Ons_ page in the EE control panel.
    - Find _Calendar_ in the list, and click the **Install** button.
  </StepMarkdown>
</VerticalStepWrapper>

:::info
#### Reserved Calendar field names
If you previously used regular ExpressionEngine fields for event dates or times before switching to Calendar, make sure those fields are not using names reserved by Calendar.

Avoid field names such as:

- `event_start_date`
- `event_start_time`
- `event_end_date`
- `event_end_time`

These names are reserved for Calendar and may cause template errors such as:

`DateTime::__construct(): Failed to parse time string (0) at position 0 (0): Unexpected character`

To resolve this, rename or remove any custom fields using reserved Calendar field names.
:::

## Updating Instructions

Calendar has its own **Update Service** built in, which means that every time there's an update available, an **Updates Available** nav item will show up in the Calendar add-on area of the EE control panel. You can then review the changelog there, or [view it here](./changelog.mdx).

<Photo
  img={require('@site/static/img/ee/calendar/v5/cp_updates.png')}
  alt="Calendar's Built in Update Service"
/>

To download the update, simply click the **Get Latest Version** button, and you'll be taken to your account area on the [EE Add-ons Store](https://expressionengine.com/add-ons/calendar) site where you can log in and download the updated package for Calendar. Then follow the steps below:

<VerticalStepWrapper>
  <StepMarkdown stepTitle="Prepare your Site">
    - Inside your EE site's `./system/user/addons/` directory, delete the `calendar` folder.
    - Inside your EE site's `./themes/user/` directory, delete the `calendar` folder.
  </StepMarkdown>
  <StepMarkdown stepTitle="Unzip the Download Package">
    - Open the **EE7** folder.
    - Copy the `calendar` folder to your EE site's `./system/user/addons/` directory.
    - Within the `themes` folder, copy the `calendar` folder to your EE site's `./themes/user/` directory.
  </StepMarkdown>
  <StepMarkdown stepTitle="Update the Add-on">
    - Go to the _Add-Ons_ page in the EE control panel.
    - Click on the **Updates** tab near the top, and find _Calendar_ in the list.
    - Click the **Update** button.
  </StepMarkdown>
</VerticalStepWrapper>

## Quick Setup

Follow the steps below to quickly get Solspace Calendar set up in minutes.

<VerticalStepWrapper>
  <StepMarkdown>
    Customize and configure Solspace Calendar settings by visiting the [Settings](../guides/control-panel.mdx#preferences) page (**Add-Ons → Calendar → Preferences**).
  </StepMarkdown>
  <StepMarkdown>
    Create Calendar field(s), calendars, and set some preferences in the _Preferences_ area of the [Control Panel](../guides/control-panel.mdx#preferences).
  </StepMarkdown>
  <StepMarkdown>
    Install the [Demo Templates](../guides/control-panel.mdx#demo-templates) (**Add-Ons → Calendar → Preferences → Demo Templates**) to get Calendar up and running on the front end with just a couple clicks!
  </StepMarkdown>
</VerticalStepWrapper>
