# Settings

AI Assistant settings let you control where the inline AI button appears and optionally assign default prompts to each field.

## Enable AI on Fields

Select which fields should display the inline AI button.

<VerticalStepWrapper>
  <StepMarkdown>
    Go to **AI Assistant → Settings**
  </StepMarkdown>
  <StepMarkdown>
    Toggle which fields should show the inline AI button
  </StepMarkdown>
  <StepMarkdown>
    Click the **Save** button
  </StepMarkdown>
</VerticalStepWrapper>

  img={require('@site/static/img/craft/ai-assistant/v1/configuration/settings-fields.jpg')}
  alt="Enable AI on Fields"
/>

## Field-Specific Prompts

You can assign a default prompt to each field so the _Generate_ modal opens with the right context.

<VerticalStepWrapper>
  <StepMarkdown>
    Go to **AI Assistant → Settings**
  </StepMarkdown>
  <StepMarkdown>
    In the **Field Prompts** table, choose a prompt for each field (or leave as _Any_)
  </StepMarkdown>
  <StepMarkdown>
    Click the **Save** button
  </StepMarkdown>
</VerticalStepWrapper>

<Photo
  img={require('@site/static/img/craft/ai-assistant/v1/configuration/settings-fields-edit.jpg')}
  alt="Field-specific Prompts"
/>

### Behavior on Entry Edit Pages

- Clicking the inline AI button will automatically select the field's assigned prompt.
- Supported field types include:
  - Craft fields: _Plain Text_, _CKEditor_, _Redactor_, _TinyMCE_, _Assets_
  - Built-in field: _Title_ (`title`)
- The _Title_ field uses the configured prompt from settings, even though it isn't a custom field.

### Notes

- **Any** means no default prompt will be selected.
- If a prompt is tied to a specific integration, that integration will be used automatically.
- Changes made to the selected prompt inside an entry are not saved back to Settings.
  - To update the default, edit it in **AI Assistant → Settings**.
