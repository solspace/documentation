# Plugin Settings

_Solspace Calendar_ allows you extra customization of your event management experience. To adjust your settings, click the **Settings** menu item while in the Calendar plugin, or go to **Settings > Plugins > Calendar** and click the settings link.

## License

- **License key** <a href="#license-key" id="license-key" class="docs-anchor">#</a>
  - This isn't a required field, but it allows you to keep track of your licenses easier. Simply enter the Calendar license key you received here.

## General Settings

<Photo
  img={require('@site/static/img/craft-calendar/v1/cp_settings-general.png')}
  alt="General Settings"
/>

- **Display Mini Calendar & Calendars list in Month/Week/Day views?** <a href="#display-mini-cal-calendars" id="display-mini-cal-calendars" class="docs-anchor">#</a>
  - Displays a side menu with mini calendar and list of calendars, allowing you to filter your event results.
- **Include Disabled Events in Month/Week/Day views?** <a href="#include-disabled-events" id="include-disabled-events" class="docs-anchor">#</a>
  - Events that are disabled will be displayed in the views with faded styles.
- **Allow new events to be created in Month/Week/Day views?** <a href="#allow-new-quick-events" id="allow-new-quick-events" class="docs-anchor">#</a>
  - Allows all users with event creation privileges to use the Quick Create event feature.
- **Restrict users to editing their own events only?** <a href="#restrict-author-edit" id="restrict-author-edit" class="docs-anchor">#</a>
  - Users (excluding Admins) with event creation privileges will only be able to edit or delete their own events.
- **Default View** <a href="#default-view" id="default-view" class="docs-anchor">#</a>
  - The default page to go to when clicking the Calendar nav item. Option are:
    - Month view
    - Week view
    - Day view
    - Events
    - Calendars
- **Hide "Install Demo" alert message?** <a href="#hide-demo-install" id="hide-demo-install" class="docs-anchor">#</a>
  - Hides the alert message suggesting to install demo templates when this setting is enabled.

## Event Settings

<Photo
  img={require('@site/static/img/craft-calendar/v1/cp_settings-events.png')}
  alt="Event Settings"
/>

- **Default Event Duration** <a href="#event-duration" id="event-duration" class="docs-anchor">#</a>
  - The default event time duration for fieldtype, in minutes. When event Start Time is selected, this number of minutes will automatically be added to that time for End Time field.
- **Time Interval** <a href="#time-interval" id="time-interval" class="docs-anchor">#</a>
  - The time picker interval for fieldtype, in minutes. The Start Time and End Time field time pickers will contain a list of times of the day by this interval.
- **Select 'All Day' by default?** <a href="#all-day-default" id="all-day-default" class="docs-anchor">#</a>
  - When enabled, when creating new events the 'All Day' option will be selected by default.
- **Date Overlap Threshold** <a href="#date-overlap-threshold" id="date-overlap-threshold" class="docs-anchor">#</a>
  - Specify the amount of hours in the next day which are still considered as those of the previous day. This prevents late night events from showing up the next day for calendars in control panel and front end.

## Guest Access

<Photo
  img={require('@site/static/img/craft-calendar/v1/cp_settings-guest.png')}
  alt="Guest Access Settings"
/>

- **Allow Guest Access on Front End?** <a href="#guest-access-allow" id="guest-access-allow" class="docs-anchor">#</a>
  - Enable this to allow guests to post events to specific calendars on the front end.
- **Allowed Calendars** <a href="#guest-access-allowed-calendars" id="guest-access-allowed-calendars" class="docs-anchor">#</a>
  - Select which calendars Guests should have access to create events for on the front end.

Allows you to install the [Demo Templates](./demo-templates.mdx) to get Calendar up and running on the front end with just a couple clicks!
