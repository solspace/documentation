# Generate Fake Data<Badge type="feature" text="New in 5.2+" />

If you ever wish to preload form fields with automatically generated fake submission data for testing purposes or other reasons, you can do so with Freeform's implementation of the [Faker](https://fakerphp.org/) library. It's already included, so you don't need to install or enable anything extra.

:::warning
Please note that any _Faker_ variables you add to forms will be visible to anyone when the form is displayed on the front end. This is intended for testing purposes only.
:::

:::tip
[Need to mass generate fake submission data for testing purposes? You can do so with the `freeform/submissions/generate` CLI command.](../../configuration/console-commands/#generate-fake-submissions)
:::

## Common Formatters
Below is a list of common formatters for using with Freeform forms. For the complete list of formatters, please see the [Faker documentation](https://fakerphp.org/formatters/).

:::info
Please note that not all _Faker_ formatters and syntax noted on the [Faker documentation](https://fakerphp.org/formatters/) will work inside Freeform.
:::

## Setup Guide

<VerticalStepWrapper>
<StepMarkdown>

Inside the form builder, click on any field you wish to set a fake value for.

</StepMarkdown>
<StepMarkdown>

Inside the **Default Value** setting for the field, enter a _Faker_ variable. This is constructed as follows:

> `{{ faker` + `.` + _formatterName_ + ` }}`

Refer to the [list of common formatters](#common-formatters) above for assistance with this.

For example, you will likely add `{{ faker.email }}` to an **Email** field, and `{{ faker.firstName }}` to a **First Name** field.

<Photo
  img={require('@site/static/img/craft/freeform/v5/developer/faker-form-builder-field.png')}
  alt="Add Faker to your form fields inside the form builder"
  scale="75"
/>

</StepMarkdown>
<StepMarkdown>

_Faker_ currently can only be applied to fields that are text/textarea input types. It's also recommended that you apply the `|raw` filter in cases where you may get values with apostrophes, such as names (e.g. `O'Conner`).

Once done, your form might look something like this:

<Photo
  img={require('@site/static/img/craft/freeform/v5/developer/faker-form-builder-overview.png')}
  alt="An overview of Faker set into default values of fields for the form"
/>

On the front end, it will render like this:

<Photo
  img={require('@site/static/img/craft/freeform/v5/developer/faker-front-end-form.png')}
  alt="A form on the front end with automatically generated test data from Faker"
/>

</StepMarkdown>
<StepMarkdown>

If you see any errors when attempting to load your form on the front end, you may need to make adjustments to your _Faker_ variables or use a different approach.

</StepMarkdown>
</VerticalStepWrapper>
